## ⚠️ Pendências / avisos importantes

### 1. Auto-deploy NÃO funciona (webhook ausente)
O projeto Vercel tem o link Git na config, **mas não há webhook no GitHub** (`gh api .../hooks` retorna vazio). Por isso `git push` **não** dispara deploy — todos os deploys são manuais via CLI.

**Deploy manual (o que uso):**
```bash
cd "APLICAÇÕES/clientes/a-formula/site-institucional"
vercel --prod --yes    # .vercel/ já está linkado ao projeto correto
```

**Para religar o auto-deploy (só o operador faz — OAuth no browser):**
claude.ai/Vercel dashboard → projeto `aformula-institucional` → Settings → Git → Disconnect → Connect Git Repository → reautorizar `Vinisal-Design/AFormula_institucional`.

### 2. Deploy CLI exige `vercel link` antes
`vercel --prod` SEM link cria projeto novo pelo nome da pasta. A pasta chama `site-institucional`, então já criou por engano o projeto `site-institucional` (prj_3P8e2Y0xHlZlnvEtKR20PqSEYASp) — candidato a remover. O `.vercel/project.json` já aponta pro projeto certo; não apagar.

### 3. Cache immutable nos assets
Trocar imagem mantendo o mesmo nome de arquivo → usuários com cache veem a versão antiga até expirar. **Ao trocar imagem, usar nome novo** (ex: `foto-v2.webp`) pra forçar atualização. Ou hard refresh (Ctrl+Shift+R) pra conferir.

### 4. Extras não aplicados (opcionais)
- Ofuscar emails texto-claro (`sac@`, `pet@aformulabr.com.br`) contra spam-bots.
- Adiar `maplibre-gl.js` (1MB, só na página encontre-uma-loja).
- CSP usa `'unsafe-inline'` (obrigatório: 808 inline styles + scripts inline herdados do Claude Design). Remover exigiria refatorar todo CSS/JS — fora de escopo.
